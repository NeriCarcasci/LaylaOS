# Chapter 5.2 — Syscall ABI and the Ring 3 Shell

Your syscall path is one of the best learning assets in this repo.

## Files

- `interrupts.cpp` / `interrupts.s` (`int 0x80` gate and dispatch path)
- `syscall.h` / `syscall.cpp`
- `shell_program.s`
- `keyboard_buffer.*`, `terminal.*`

## ABI Entry

- IDT vector `128` is set with type attr `0xEF` (user-callable trap gate).
- User program executes `int $0x80` with syscall number in `EAX`.
- Kernel receives a `SyscallFrame` and writes return value back to frame `eax`.

## Implemented Syscall Set (Current)

Notable numbers in your dispatch:

- `1`  exit
- `3`  read
- `4`  write
- `5`  open (used for touch/create)
- `6`  unlink
- `7`  waitpid
- `11` exec
- `20` getpid
- `39` mkdir
- `50-55` network send/recv/connect/close
- `88` reboot
- `89` ls
- `90` cp
- `91` mv

## Safety Model

You validate user pointers with explicit range checks (Ring 3 region + shared net page).

This is critical because kernel dereferences user-provided addresses.

## Blocking Model

Blocking syscalls set process state to BLOCKED and use `hlt` loops while interrupts drive wakeup:

- keyboard input wait (`wait_pid = -2`)
- waitpid child completion
- UDP/TCP receive timeouts with PIT ticks
- TCP connect timeout

It’s simple but very teachable.

## Ring 3 Shell

`shell_program.s` parses command prefixes and maps each command to syscall numbers.

Commands include:

- filesystem (`ls`, `rm`, `mkdir`, `touch`, `cp`, `mv`)
- info (`help`, `uname`, `whoami`, `meminfo`)
- networking (`ping`, `nc`)
- control (`reboot`, `exit`)
