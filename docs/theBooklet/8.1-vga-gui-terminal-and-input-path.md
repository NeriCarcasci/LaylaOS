# Chapter 8.1 — VGA, GUI, Terminal, and Input Path

This chapter teaches how your visible UI is built and fed by interrupts.

## Files

- `vga.*`
- `gui.*`
- `terminal.*`
- `keyboard.*`
- `keyboard_buffer.*`
- `mouse.*`
- `boot_anim.*`

## Rendering Stack

### VGA layer

`VGA` supports:

- text mode writes to `0xB8000`
- mode 13h graphics (`0xA0000` framebuffer)
- primitive drawing (`DrawPixel/DrawRect/DrawChar`)

### GUI layer

- `Desktop` manages windows and cursor
- `Window` draws title bar and child widgets
- `Widget` base class routes input events

### Terminal layer

`Terminal` is a `Window` with 40x23 char grid, internal input line buffer, and shell output rendering.

## Input Flow

### Keyboard IRQ path

1. IRQ1 handler reads scancode.
2. Applies shift/caps logic.
3. Produces char.
4. In graphics mode: forwards to active desktop/window/terminal.
5. Otherwise enqueues to `KeyboardBuffer` and prints in text mode.

### Mouse IRQ path

1. IRQ12 accumulates 3-byte PS/2 packet.
2. Deltas converted to desktop movement.
3. Cursor redraw handled by desktop.

## Shell Input Bridge

`Terminal::OnKeyPress('\n')` pushes completed line into `KeyboardBuffer` and wakes waiting reader process. `sys_read` consumes from this buffer.

So GUI input and syscall read are connected by one queue.

## Boot Visuals

`BootAnimation()` runs after entering graphics mode and before desktop setup, including custom palette and ring animation.
