# Chapter 3.2 — `kernelMain` Initialization Walkthrough

This is the most important chapter for understanding your OS as a system.

## File to Trace

- `kernel.cpp`

## Exact Initialization Order (Your Code)

1. Build `TaskStateSegment` and `GlobalDescriptorTable`
2. Parse multiboot memory map, choose largest usable region above 2 MiB
3. Cap heap window to first 16 MiB
4. Reserve top 512 KiB of that region for PMM bitmap-backed frame pool
5. Initialize heap allocator (`MemoryManager`) and global `new/delete`
6. Initialize PMM and mark PMM pool as free frames
7. Initialize paging and enable kernel directory
8. Initialize VGA text and print boot line
9. Build interrupt manager, keyboard buffer, keyboard, mouse
10. Enumerate PCI and activate drivers
11. Discover ATA driver and mount FAT32 from partition 0 if valid
12. Discover RTL8139 and wire ARP/IPv4/ICMP/UDP/TCP handlers
13. Copy shell program blob to `0x00400000`
14. Clear network shared page (`NET_SHARED_BASE`)
15. Program PIT divisor (`11932`) for scheduler ticks
16. Enter graphics mode, set custom palette, run boot animation
17. Create `Desktop` + `Terminal`, set active desktop/window
18. Create scheduler and shell process
19. Activate interrupts and spin forever

## Why This Order Is Correct

- Interrupts are enabled only after critical subsystems are ready.
- Scheduler starts only after process image exists.
- Drivers are activated before filesystems/network try to use them.

## Key Global Contracts

- `global_fat32` is optional (depends on disk/partition/mount success)
- `global_nic`, `global_ipv4`, `global_udp`, `global_tcp` exist only if NIC found
- Syscall layer checks these pointers before use

## Learning Exercise

Draw this flow on one page as a dependency graph. If you can reorder nodes and predict failures, you understand boot runtime composition.
