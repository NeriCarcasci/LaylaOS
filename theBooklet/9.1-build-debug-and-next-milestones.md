# Chapter 9.1 — Build, Debug, and Next Milestones

You now have a working educational OS. This chapter helps you keep learning without losing stability.

## Build + Run Baseline

Core commands:

- `make mykernel.iso`
- `qemu-system-i386 -cdrom mykernel.iso`

With NIC and disk for full-stack testing:

- add RTL8139 device + user networking
- add IDE disk image for ATA/FAT32

## Debug Strategy That Works

Use a strict one-change loop:

1. change one function or one constant
2. rebuild + boot
3. observe exact difference
4. revert if unclear

Good debug anchors in this codebase:

- early `VGA::Print` in boot path
- scheduler PID prints on tick
- syscall dispatch prints by number
- network state prints in TCP transitions

## Common Failure Classes

- Wrong interrupt setup (missing EOI / bad gate attrs)
- Wrong stack frame assumptions between asm and C++
- Missing pointer validation for user buffers
- Page mapping mistakes causing immediate fault/halt
- Device timing assumptions (ATA/PS2)

## Suggested Learning Milestones

### Milestone A — Reliability

- add more defensive checks + debug output
- add simple kernel panic utility with register dump

### Milestone B — Userland evolution

- load user programs from FAT32 by path
- support multiple shell binaries

### Milestone C — Memory hardening

- lazy page allocation on page fault
- stronger user/kernel memory separation rules

### Milestone D — Network maturity

- UDP sockets with multiple local ports
- improved TCP retransmission + timeout control

### Milestone E — Filesystem maturity

- long filename support
- path normalization and better errors

## Final Advice

Keep this project “small but correct.”

A kernel you can reason about is worth more than a kernel with many half-working features.
