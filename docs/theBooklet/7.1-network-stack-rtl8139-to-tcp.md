# Chapter 7.1 — Network Stack: RTL8139 to TCP

This chapter follows packets through your entire network stack.

## Files

- `rtl8139.*`
- `ethernet.*`
- `arp.*`
- `ipv4.*`
- `icmp.*`
- `udp.*`
- `tcp.*`
- `syscall.cpp` + `net_shared.h` + `net_hooks.h`

## Bring-Up Sequence in `kernelMain`

1. PCI discovers RTL8139 and driver activates.
2. NIC MAC is read and RX/TX buffers configured.
3. ARP handler created with local IP `10.0.2.15`.
4. IPv4 handler created and registered on ethertype IPv4.
5. ICMP/UDP/TCP handlers registered by protocol number.
6. Initial ARP request sent to `10.0.2.2`.

## Layer Responsibilities

### RTL8139

- Handles IRQ-driven RX
- Copies RX frame to Ethernet dispatcher
- Sends TX frames via descriptor ports

### Ethernet

- Demultiplexes by ethertype (`0x0806`, `0x0800`)

### ARP

- caches IPv4→MAC mappings
- replies to ARP requests targeting local IP
- resolution required before IPv4 send

### IPv4

- validates header/checksum
- accepts packets only for local IP
- demuxes protocol to ICMP/UDP/TCP

### ICMP

- echo request → echo reply

### UDP

- port-based receive filtering
- send path builds UDP header

### TCP

- minimal state machine for connect/send/close
- handles SYN/SYN-ACK/ACK handshake and simple FIN flow

## Syscall Networking Bridge

`NET_SHARED_BASE` page is a shared ring3/kernel buffer for payload transfer.

Syscalls:

- `50` UDP send
- `51` UDP recv
- `52` TCP connect
- `53` TCP send
- `54` TCP recv
- `55` TCP close

Shell commands `ping` and `nc` use this interface.
