# Chapter 0.1 — How to Learn This Codebase Effectively

This booklet is designed to teach you **from your actual implementation** in Layla/Frodo OS, not from a generic textbook kernel.

## How to Use This Booklet

For each chapter, do this loop:

1. Read the chapter once quickly.
2. Open the referenced source files and trace the control flow.
3. Rebuild and run once.
4. Change one tiny thing and re-test.
5. Write down what broke and why.

That loop is where the real learning happens.

## Your Kernel Learning Order

Use this order exactly:

1. **CPU setup**: GDT → IDT/PIC → TSS
2. **Hardware I/O**: Port I/O → keyboard/mouse → PCI
3. **Boot/runtime wiring**: loader, linker, kernel init order
4. **Memory**: heap allocator → PMM → paging
5. **Execution model**: process context → scheduler → syscalls
6. **Storage**: ATA → MBR → FAT32
7. **Network**: RTL8139 → Ethernet → ARP/IPv4 → ICMP/UDP/TCP
8. **UI path**: VGA → desktop/window/terminal → input routing

## Ground Rules (Important)

- Never change 5 subsystems at once.
- Keep experiments small and reversible.
- If behavior is unclear, instrument with tiny prints.
- Prefer proving assumptions over guessing.

## What “Understanding” Looks Like

You understand a subsystem when you can:

- Explain its data flow in 30 seconds.
- Name the key structs/classes/functions.
- Predict one failure mode before running.
- Modify it safely without random side effects.

## Mini Practice Routine

For each chapter, answer these:

- What triggers this code path?
- What state does it read and write?
- What are the invariants?
- What would crash if this step were removed?

If you can answer those from code, you’ve learned it.
