# Chapter 6.1 — Storage Stack: ATA → MBR → FAT32

This chapter explains your full disk read/write pipeline.

## Files

- `ata.h` / `ata.cpp`
- `mbr.h` / `mbr.cpp`
- `fat32.h` / `fat32.cpp`
- `kernel.cpp` (mount flow)
- `syscall.cpp` (file-related syscalls)

## Layer 1: ATA (PIO)

`ATADriver` talks to primary IDE ports (`0x1F0` family).

Key operations:

- `Identify()` via command `0xEC`
- `Read28()` via command `0x20`
- `Write28()` via command `0x30`

Synchronization uses `WaitBSY()` and `WaitDRQ()` polling.

## Layer 2: MBR

`MBRParser::Read()` reads LBA 0 and validates signature `0xAA55`.

Then partition entries provide:

- type
- start LBA
- sector count

Kernel chooses partition index 0 if available.

## Layer 3: FAT32

Mount computes:

- FAT region LBA
- data region LBA
- cluster size

Current capabilities include:

- file read (`ReadFile`)
- directory listing (`ListDirectory*`)
- file write/create/delete
- directory create

Implementation uses 8.3 short-name conversion and cluster chain traversal.

## Syscall Integration

Filesystem syscalls call FAT32 methods directly after pointer checks.

Examples:

- `ls` syscall 89
- `touch` via open syscall 5 + create flag
- `mkdir` syscall 39
- `rm` syscall 6
- `cp`/`mv` syscalls 90/91

## Important Constraints

- 512-byte sector assumption
- no long filename support (LFN entries skipped)
- simple path traversal model
