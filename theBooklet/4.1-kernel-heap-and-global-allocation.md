# Chapter 4.1 — Kernel Heap and Global `new/delete`

Your kernel has a simple first-fit heap allocator used by C++ `new`.

## Files

- `memorymanagement.h` / `memorymanagement.cpp`
- `allocator.h` / `allocator.cpp`
- `kernel.cpp` (initialization)

## Data Structure

`MemoryChunk` fields:

- `prev`, `next`
- `allocated`
- `size`

All chunks are linked in one list inside the selected heap region.

## Allocation Strategy

`malloc(size)` scans linearly for first free chunk with enough size.

If larger than needed, it splits:

- allocated part keeps requested size
- remainder becomes new free chunk

This is classic first-fit with split.

## Free Strategy

`free(ptr)` marks chunk free, then coalesces:

- forward merge with next free chunk
- backward merge with previous free chunk

So fragmentation is reduced over time.

## Global C++ Integration

`allocator.cpp` sets `active_memory_manager` and routes:

- `operator new`
- `operator delete`
- array variants

to heap allocator functions.

This means most dynamic kernel objects (drivers, handlers, processes) use this heap.

## Limits in Current Design

- No alignment-aware API
- No out-of-memory diagnostics
- O(n) allocation scan
- Single global heap, no locking

All acceptable for this educational single-core kernel.
