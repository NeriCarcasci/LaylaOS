# Chapter 4.2 — Physical Memory Manager and Paging

This chapter explains your frame allocator and page table model.

## Files

- `pmm.h` / `pmm.cpp`
- `paging.h` / `paging.cpp`
- `kernel.cpp` (init order)

## PMM Model

`PhysicalMemoryManager` uses a bitmap:

- 1 bit per 4 KiB frame
- bit set = used
- bit clear = free

Initialization marks everything used, then selected range is explicitly freed (`MarkFree`).

### Core operations

- `AllocFrame()` finds first clear bit and marks used
- `FreeFrame(addr)` clears bit
- `MarkUsed/MarkFree(addr,len)` bulk region updates

This makes frame ownership explicit and deterministic.

## Paging Model

`PageDirectory` holds:

- physical page directory frame (`phys_dir`)
- virtual pointers to page tables (`page_tables[1024]`)

### Kernel directory

`CreateKernel()` identity maps `0x00000000`–`0x01000000` as present+write.

### User directory

`CreateUser()` starts by copying kernel mappings, then user regions are mapped explicitly per process.

## CR3/CR0 Activation

`PagingManager::Enable(dir)`:

- loads `CR3` with directory frame
- sets paging + protection bits in `CR0`

`SwitchDirectory(dir)` replaces `CR3` on context switch.

## Page Fault Handling

Interrupt manager special-cases vector 14:

- reads `CR2` fault address
- calls `PagingManager::HandlePageFault`
- current implementation prints and halts

Great for learning: failures are loud and immediate.
